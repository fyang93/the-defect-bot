# i18n 原则

本项目中的 i18n 只保留 **UI 相关文本**。

## 保留范围

允许放进 i18n 的内容：

- 按钮文字
- 菜单标题
- 翻页、返回、确认、取消等导航词
- callback alert / toast
- 简短权限提示
- 简短系统状态提示
- 命令菜单名称
- 面板型字段标签

这些文本的特点是：

- 稳定
- 短
- 明确属于界面壳层
- 不依赖模型创作

## 不应放进 i18n 的内容

以下内容默认不应由 i18n 模板生成：

- 提醒发送文案
- 面向用户的自然语言回复
- 日程/提醒/事件的自然语言总结
- 由代码拼出的完整说明句
- 各种“every ... at ...”这类人类表述模板
- reminder offset、weekday、ordinal、lunar leap policy 这类仅为拼自然语言句子服务的词表

这些内容应尽可能改为：

- 代码提供结构化事实
- 模型生成最终用户可见文本

## 代码职责边界

- 代码负责：时间计算、状态判断、结构化事实、UI 壳层
- 模型负责：用户可见自然语言

## 审查标准

看到新的 i18n key 时，先问：

> 这是不是 UI 壳层文字？

如果不是，就优先不要放进 i18n。

再问：

> 这段文本是不是更适合由模型根据事实生成？

如果是，就应从 i18n/模板代码中移出。

## 对现有系统的落地要求

1. 删除 reminder / schedule 的自然语言句式模板。
2. 保留 schedule 菜单、按钮、列表页、确认框等 UI 文本。
3. 非 UI 的用户可见输出尽量走模型。
4. 不再新增“代码拼自然语言”的多语言模板层。
