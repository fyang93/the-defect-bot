# Form Filling Experience

Use this alongside the format-specific skill (`xlsx`, `docx`, `pdf`, etc.). This skill is only about form-filling judgment, not file editing mechanics.

## Judgment rules

- Treat a form as an official template: fill answers, do not redesign the form.
- Do not overwrite already-filled fields unless the user explicitly asks.
- Prefer leaving uncertain fields blank over guessing. Ask only for fields that are required and not in memory.
- If the identity of the form subject/applicant is unclear, confirm it before using person-specific memory.
- Distinguish durable facts from one-off application choices:
  - durable: phone number, address, passport/residence-card info, exact graduation dates
  - one-off: desired status/period, application date, whether to hand-circle an option
- If the form has applicant/user sections and organization/third-party sections, fill only the applicant/user side unless instructed otherwise.
- For choice fields, follow the form’s own convention. If the form expects selecting between blank and filled boxes, use the filled box it already supports; do not invent a nicer-looking checkbox.
- For printed circle choices such as `有・無`, do not add extra markers when the user wants to circle by hand.
- Do not add postal codes to address fields unless the label or user asks for postal code.
- Leave signature fields blank when the user intends to sign by hand.
- Before finalizing, re-check the latest user correction against the document; the latest instruction wins over earlier memory or guesses.

## Memory use

- Load `memory` first and reuse known facts.
- Store newly provided reusable facts in the relevant person note when useful later.
- Do not store one-off form choices unless the user asks.
